I recalled seeing [Lit.dev](https://lit.dev/) few months ago, I didn't know much about its use case at a time. But now when I was trying to add a kbar alternative to the normal html, I was struggling a bit, until I see [ninja-keys](https://github.com/ssleptsov/ninja-keys) which built with Lit makes it possible to integrate with any framework. 

I bet it'd be cool to create components or contributing existing components and moving 'em into framework agnostic would definitely save tons of trouble. 

Some of my current favorite frontend libraries:

1. [Shadcn/ui](https://ui.shadcn.com/docs/components/data-table)

2. [Flowbite](https://flowbite.com/)

Advantages:

1. Framework agnostic

2. Able to embedded different type of frontend library. (Like using chakra-ui with bulma and the others).
